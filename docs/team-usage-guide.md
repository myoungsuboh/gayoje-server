# 팀(Team) 사용 가이드

> **플랜 안내**: 팀 기능은 **Pro 플랜(유료)** 전용입니다.  
> 팀장(owner)과 초대받는 멤버 모두 Pro 플랜이어야 합니다.  
> 무료 계정으로 팀 기능에 접근하면 `402 Payment Required` 오류가 반환됩니다.

---

## 1. 팀 만들기

1. 우측 상단 프로필 아이콘 → **프로필 설정** 페이지로 이동
2. **팀 관리(Team Management)** 카드에서 **+ 새 팀 만들기** 클릭
3. 팀 이름 입력 후 생성 → 본인이 자동으로 **owner** 역할로 등록됨

> 팀 이름은 내부 식별자로 사용되며 이후 변경하기 어려우므로 신중히 입력하세요.

---

## 2. 멤버 초대

1. 프로필 → **팀 관리** 카드 → 해당 팀의 **멤버 초대** 버튼 클릭
2. 초대할 사람의 **가입 이메일** 입력
3. 역할 선택 (admin 또는 member) → 초대 전송
4. 상대방이 Pro 플랜이어야 초대를 수락할 수 있음

### 역할 권한

| 역할   | 멤버 초대 | 멤버 역할 변경 | 프로젝트 생성/삭제 | 모든 작업 |
|--------|-----------|---------------|-------------------|-----------|
| owner  | ✅        | ✅            | ✅                | ✅        |
| admin  | ✅        | ✅ (member만) | ✅                | ✅        |
| member | ❌        | ❌            | ✅                | ✅        |

---

## 3. 팀 프로젝트 만들기

팀 프로젝트는 **일반 개인 프로젝트와 별도 공간에 격리**됩니다.  
"새 프로젝트" 버튼은 항상 개인 프로젝트를 만들므로 팀 프로젝트는 아래 흐름을 사용하세요.

### 신규 팀 프로젝트 생성

1. **홈(대시보드)** 페이지로 이동
2. 팀 섹션(예: `TEAM · MY TEAM`)의 **+ 새 팀 프로젝트** 버튼 클릭
3. 팝업에서 **프로젝트 이름**을 입력하고 **만들고 회의록 작성** 클릭
4. 팀 컨텍스트가 활성화된 채 **Plan(/plan)** 페이지로 이동 (헤더에 팀 배지 표시)
5. 회의록을 작성하고 저장하면 해당 프로젝트가 팀 공간에 정식 등록됨

> **왜 이름을 먼저 받나요?** 일반 "+ 신규" 입력 방식은 개인 프로젝트 전용이라
> 팀 컨텍스트를 해제합니다. 팀 프로젝트는 이름과 팀 컨텍스트를 함께 설정해야
> 하므로 전용 팝업에서 이름을 받습니다.

> 헤더 상단의 **팀 배지**(`TEAM {팀 이름}`)가 표시되어 있으면 팀 컨텍스트가 활성화된 상태입니다.

### 팀이 처음 생성되었을 때(프로젝트 0개)

홈 → 팀 섹션에 "아직 팀 프로젝트가 없어요" 메시지와  
**{팀 이름} 첫 프로젝트 만들기** 버튼이 노출됩니다.  
버튼을 누르면 위와 동일한 흐름으로 진행됩니다.

---

## 4. 기존 팀 프로젝트 전환

1. **홈** 페이지의 팀 섹션에서 원하는 팀 프로젝트 카드 클릭
2. 헤더 배지가 해당 팀으로 바뀌며 모든 페이지에서 팀 컨텍스트가 유지됨
3. 개인 프로젝트로 돌아가려면 **MY PROJECTS** 섹션에서 개인 프로젝트를 클릭 → 팀 컨텍스트 자동 해제

---

## 5. 팀 컨텍스트 배지 (헤더)

헤더 상단에 `팀: {팀 이름}` 배지가 표시되면 현재 팀 프로젝트 작업 중입니다.  
배지 클릭 → 팀 컨텍스트 해제 (개인 모드로 전환).

| 상태         | 배지     | API team_id |
|--------------|----------|-------------|
| 팀 프로젝트  | 표시됨   | 자동 첨부   |
| 개인 프로젝트 | 미표시   | 미첨부      |

---

## 6. 격리 범위 — 팀 vs 개인

팀 프로젝트는 동일한 이름의 개인 프로젝트와 **완전히 분리**됩니다.

| 데이터           | 팀 프로젝트      | 개인 프로젝트   |
|------------------|------------------|-----------------|
| 회의록 / CPS / PRD | 팀 전용 저장 공간 | 개인 저장 공간  |
| 설계 (SPACK/DDD/Arch) | 팀 전용        | 개인 전용       |
| Lint 결과         | 팀 전용          | 개인 전용       |
| Lineage          | 팀 전용          | 개인 전용       |
| OAuth 토큰       | 요청자 본인 토큰 사용 | 동일          |

`my-app` (개인) ≠ `my-app` (팀) — 서버에서 네임스페이스 키로 자동 분리됩니다.

---

## 7. 팀원이 같은 프로젝트에 접근하려면

모든 팀원은 동일한 팀 ID를 공유하므로, 프로젝트 이름만 같으면 동일한 프로젝트 데이터를 조회·편집할 수 있습니다.

팀원이 홈 → 팀 섹션에서 같은 프로젝트를 클릭하면 자동으로 공유 공간으로 진입합니다.

---

## 8. 주의사항

- **팀 프로젝트를 개인 프로젝트로 이전하거나 그 반대는 지원되지 않습니다.**
- 팀에서 탈퇴하면 해당 팀 프로젝트에 접근할 수 없게 됩니다 (데이터는 유지됨).
- 프로젝트 생성 후 팀 소속 변경이 불가하므로 처음부터 팀 컨텍스트를 활성화한 뒤 만드세요.
- Pro 플랜 만료 시 팀 기능이 비활성화됩니다 (데이터는 유지).
