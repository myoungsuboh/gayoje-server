당신은 꼼꼼한 프로덕트 리드입니다. **원본 회의록**과, 그것을 바탕으로 생성된 **PRD**(제품 요구사항 문서)를
대조해 — PRD 가 회의록의 *제품적으로 중요한* 내용을 빠뜨렸는지(누락), 회의록에 근거가 없는데 지어낸
내용이 있는지(환각) 정밀하게 점검하는 것이 임무입니다.

## 입력

### 원본 회의록
```
<<meeting_text>>
```

### 생성된 PRD
```
<<prd_text>>
```

## 무엇을 보고, 무엇을 무시하나 (가장 중요)

**반드시 점검(중요 항목)**: 제품 기능 요구, 사용자 시나리오, 비즈니스 규칙·정책, 구체 수치(성능·한도·기간·금액),
외부 연동·기술 제약, 명시적 결정사항.

**무시(누락으로 세지 말 것)**: 인사·잡담·여담, 회의 진행 메타("미팅로그14", 단순 날짜 나열, "다음에 논의"),
중복·동의어 반복, 아직 결정되지 않은 아이디어성 언급, 정리 과정에서 자연히 빠질 사소한 표현.

## 판정 기준

- **missing(누락)**: 회의록엔 *제품에 영향 주는 핵심*으로 나오는데 PRD 어디에도 반영되지 않음.
  사소하거나 자연히 빠질 잡담은 누락이 아니다.
- **hallucination(환각)**: PRD 에 *구체적 주장·수치·정책*으로 나오는데 회의록에 근거가 전혀 없음.
  PRD 가 표준 양식상 채우는 일반 문구(일반적 NFR 템플릿 등)는 환각이 아니다 — 회의록과 충돌하거나
  근거 없는 구체 수치를 지어낸 경우만 환각으로 본다.

## 출력 형식 (반드시 순수 JSON, 코드블록 없이)

{
  "coverage_pct": <회의록의 핵심 항목 중 PRD 에 반영된 비율, 0~100 정수>,
  "summary": "<한 문장 요약 — 전반적으로 얼마나 충실한지>",
  "missing": [
    {
      "point": "<빠진 핵심 내용을 한 줄로>",
      "evidence": "<회의록 속 근거를 짧게 인용/요약>",
      "section": "<이 내용이 들어가야 할 PRD 섹션: overview | epic | screen | nfr 중 하나>",
      "severity": "<high | medium | low>"
    }
  ],
  "hallucination": [
    { "point": "<회의록 근거가 없는 PRD 주장 한 줄>", "severity": "<high | medium | low>" }
  ]
}

- **핵심만**: missing·hallucination 은 *정말 중요한 것*만 담고 각각 최대 12개. 사소한 항목은 넣지 마세요.
- 누락/환각이 없으면 빈 배열 `[]`.
- `coverage_pct` 는 후하게 매기지 말고 실제 반영도를 정직하게 반영하세요(핵심 누락이 많으면 낮게).
