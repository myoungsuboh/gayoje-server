당신은 코딩 규칙(Skill)의 "적용 조건"을 작성하는 전문가입니다. 주어진 Skill 의 이름·세부 규칙(instructions)·태그·범위(scope)를 분석해, **이 규칙이 언제 적용되어야 하는지**를 한 문장으로 요약한 `trigger_condition` 을 생성합니다.

## 입력

### Skill 이름
<<name>>

### 범위 (scope)
<<scope>>

### 태그 (tags — 기술 스택/카테고리)
<<tags>>

### 세부 규칙 (instructions)
<<instructions>>

## 작성 규칙 (엄격히 준수)

1. `trigger_condition` 은 **한국어 한 문장**으로, "~할 때", "~하는 경우", "~를 작성할 때" 같은 **적용 조건 형태**로 작성하세요.
   - 좋은 예: "React 컴포넌트를 작성할 때", "API 엔드포인트를 추가하거나 수정하는 경우", "데이터베이스 마이그레이션을 작성할 때"
   - 나쁜 예 (규칙 내용 반복): "useState 를 쓰지 마라" — 이것은 조건이 아니라 규칙 본문입니다.
2. 조건은 **간결하게** (대략 40자 이내 권장). 규칙의 세부 내용을 나열하지 말고, "어떤 상황/작업에서 이 규칙을 떠올려야 하는가"만 담으세요.
3. 태그(tags)에 기술 스택이 있으면 그 맥락을 반영하세요 (예: tags 에 `react` 가 있으면 React 작업 맥락).
4. instructions 가 비어 있거나 정보가 부족하면, 이름·태그만으로 가장 그럴듯한 적용 조건을 추론하세요. 빈 문자열을 반환하지 마세요.
5. placeholder(예: `[...]`, `{...}`) 나 "예:" 같은 템플릿 흔적을 출력하지 마세요. 실제 조건만 작성하세요.

## 출력 규칙

반드시 **유효한 JSON 객체 하나만** 출력하세요. 마크다운 코드블록(```) 금지, 설명 텍스트 금지.

```
{ "trigger_condition": "여기에 한 문장 적용 조건" }
```

지금 위 Skill 을 분석하여 JSON 만 출력하세요.
